# Moonlight v0.3 — 계획 (실제 구현 착수)

> v0.1·v0.2 의 목표를 계승. 이번 버전에서 **프로토타입을 실제 앱으로** 만들고,
> **`/Applications/claude-moonlight.app` 실행 → 자동으로 컨테이너 기동 → 앱 사용** 흐름을 구현한다.

## 한 줄 요약
영어를 몰라도 논문을 편하게 읽는 **macOS 앱**. 좌측 원문 / 우측 한국어 번역 2단 대조,
문장 hover 시 반대편 동일 문장이 **저채도 파랑 형광펜**으로 강조.

---

## A. 확정된 UI (프로토타입 → 실제 뷰어로 승격)
`materials/ui-prototype/index.html` 에서 검증한 기능을 그대로 제품 뷰어로 가져간다.
- **반반 2단 레이아웃** (원문 EN 좌 / 한국어 우) + `반반·원문·번역` 보기 토글.
- **형광펜 hover 연동** — 한쪽 문장 hover → 양쪽 동일 `sid` 문장 강조.
  hover 한 쪽은 진한 파랑(+테두리), 반대편은 연한 파랑(방향 표시). 저채도.
- **집중(Focus) 모드** — 나머지 문장 디밍, 가리킨 문장쌍만 또렷.
- **단어쌍 더블클릭** — 문장 위에서 단어를 더블클릭하면 그 단어와 한국어 대응어가 양쪽에서
  **더 밝은 색(노랑)** 으로 강조되어 단어 뜻만 빠르게 학습. 파이프라인이 문장별 `wp`(단어쌍) 산출.
- (읽기 단계 3-pass 기능은 제거 — 기본 **전체(정독) 표시**.)
- **테마** light / dark (흑/백).
- **목차 아웃라인**(양쪽 동시 점프), **스크롤 싱크**(퍼센트), **읽기 진행바**.
- **문장 클릭 → 양쪽 정렬 점프**, **용어 점선+hover 툴팁**.
- figure/table/수식: **양쪽 동일 위치·미번역(이미지)**, 캡션만 번역. 수식 MathJax.
- 타이포: 한국어 행간 ~1.78 / 영문 ~1.62, ~62ch, 좌측정렬, Noto Sans/Serif KR.

## B. 데이터 형식 — 정렬된 `paper.json` (핵심 변경)
참조 레포는 단일 `paper.html` 을 만들었지만, 2단 대조 + hover 연동을 위해
**문장 단위로 정렬된 JSON** 을 산출물로 삼는다. (프로토타입의 `PAPER` 구조와 동일)
```json
{
  "id":"arxiv-XXXX", "titleEn":"", "titleKo":"", "meta":"저자 · 학회 · arXiv",
  "blocks":[
    {"type":"h2","sid":"h-1","secId":"sec-1","en":"1. Introduction","ko":"1. 서론"},
    {"type":"p","pass":"read","ss":[{"sid":"s1","en":"...","ko":"..."}]},
    {"type":"figure","id":"fig-1","img":"figures/fig-1.png","capEn":"","capKo":""},
    {"type":"table","id":"tbl-1","capEn":"","capKo":"","head":[],"rows":[[]]},
    {"type":"eq","tex":"$$...$$","pass":"deep"}
  ]
}
```
- `sid` 가 좌/우를 잇는 키. `pass` 가 3-pass 필터 기준.
- 뷰어(`static/index.html`)는 `/papers/<id>/paper.json` 을 fetch 해 렌더.

## C. 번역 엔진 — 참조 레포 그대로 (v0.2 약속 유지)
- **PTY 인터랙티브 `claude` 워커**로 구동. **`claude -p` 금지.**
  입력 제출은 텍스트 → 짧은 대기 → 별도 `\r` (`worker.submit`).
- 파이프라인(`process_paper_prompt.md`): PDF 다운 → 페이지 읽기 → **문장 단위 영/한 정렬로 `paper.json` 작성**
  → figure/table 이미지 추출(`pdftoppm`+PIL crop) → 검증 → `DONE`/`FAILED`.
- **모델/품질: Opus 4.8 + effort `high` 로 고정** (`WORKER_MODEL=claude-opus-4-8`,
  `WORKER_EFFORT=high` → 워커가 세션에서 `/effort high` 설정). 잡 큐(atomic rename), 상태 파일 분리 계승.
- 달라진 점: 산출물이 `paper.html` → `paper.json`(정렬). 엔진 자체는 동일.

## D. 배포 — 앱 실행 시 컨테이너 자동 기동 (★ v0.2 와 달라진 핵심)
> **요구사항 변경**: "항상 백그라운드/부팅 자동 시작" 이 아니라,
> **앱을 실행할 때만** 컨테이너가 떠서 쓸 수 있게 한다(on-demand).

### 흐름
```
/Applications/claude-moonlight.app  더블클릭
  → (launcher) docker compose up -d   # 이미 떠 있으면 재사용
  → 헬스체크: http://127.0.0.1:8090 응답 대기
  → UI 열기 (Chrome app 모드 우선, 없으면 기본 브라우저)
```
- **LaunchAgent(부팅 자동 시작) 미설치** — 앱을 켤 때만 컨테이너가 뜬다.
- 종료 정책: **앱을 종료하면 컨테이너 + 호스트 워커가 함께 정지**(`applicationWillTerminate` →
  `scripts/moonlight-stop.sh`). 완전한 on-demand. 번역 진행 중 종료 시 해당 잡은
  `jobs/processing` 에 남았다가 다음 실행 때 `WORKER_REQUEUE` 로 자동 재개된다.

### 컨테이너 구성 (결정)
- 단일 이미지에 **server + worker + 파이프라인 도구(poppler) + claude CLI** 포함.
- **claude 인증**: 호스트의 claude 설정 디렉터리(`~/.claude` 등)를 컨테이너에 **마운트**해 로그인 상태 공유.
  (컨테이너 내 claude 인증이 매끄럽지 않을 경우의 대안: 워커만 호스트에서 돌리는 하이브리드 — 리스크로 기록.)
- 포트 8090 노출(8080 은 claude-web-terminal 예약). `papers/` 는 볼륨으로 호스트에 영속.
- 전제(v0.2 유지): 사용자 맥북에 **Docker 실행 중 + claude CLI 로그인** 되어 있음.

## E. 산출물 / 프로젝트 구조
```
claude-moonlight/
├── app/
│   ├── server.py                 # aiohttp 서버 (목록/추가/상태/정적, 참조 app.py 기반)
│   ├── worker.py                 # PTY 인터랙티브 claude 워커 (참조 worker.py 기반)
│   ├── process_paper_prompt.md   # 파이프라인 지침 (paper.json 정렬 산출로 확장)
│   ├── validate_paper.py         # paper.json 스키마 검증
│   ├── static/index.html         # 대시보드 + 2단 뷰어 (프로토타입 승격)
│   └── sample/                   # 즉시 데모용 정렬 샘플 paper.json
├── Dockerfile
├── docker-compose.yml
├── requirements.txt
├── scripts/
│   ├── moonlight-launch.sh       # .app 이 호출: compose up -d + 헬스체크 + UI 열기
│   ├── moonlight-stop.sh         # compose down
│   └── make-app.sh               # /Applications/claude-moonlight.app 생성
└── README.md
```

## F. 작업 순서 (이번 세션)
1. requirements / Dockerfile / docker-compose 작성.
2. `server.py` (참조 app.py 슬림화: 목록·추가·상태·정적 + 샘플 프리로드).
3. `static/index.html` 뷰어 = 프로토타입을 `paper.json` fetch 형태로 일반화 + 대시보드.
4. 샘플 `paper.json` 1건 → 컨테이너 없이도 즉시 열람 가능하게.
5. `worker.py` + `process_paper_prompt.md` + `validate_paper.py` (엔진은 참조 계승, 산출물만 JSON).
6. `scripts/moonlight-launch.sh` / `make-app.sh` → `/Applications/claude-moonlight.app` 생성.
7. 검증: 서버 직접 구동 → 뷰어 렌더 확인. (Docker 데몬 켜지면 compose 빌드/기동 검증.)

## G. 검증 메모 / 한계
- 현재 환경에서 **Docker 데몬 미실행** → compose 빌드·기동은 사용자가 Docker 를 켠 뒤 검증 필요.
  그 전까지는 server.py 직접 실행 + 프리뷰로 UI/데이터 흐름을 검증한다.
- 실제 번역(PTY claude) 정확도는 라이브 실행 시 확인. v0.3 은 파이프라인 골격 + 산출물 형식 확정까지.
