## 1. 단어쌍(wp) 검증 단계 추가 — "생성 → 검증" 파이프라인
**문제**: 지금도 단어 더블클릭 시 한국어 뜻이 안 뜨는 문장이 많다(`wp` 누락·부정확).
프롬프트에 "모든 문장 wp 필수"로 강화했지만 모델이 여전히 빠뜨린다.

**원하는 동작**: 번역 **생성 단계 이후에 별도의 "검증 단계"** 를 둔다.
- 검증 단계에서 에이전트가 **각 단어를 실제로 더블클릭하듯** 점검 — 즉 문장의 내용어마다
  `wp` 에 대응하는 한국어 뜻이 실제로 나오는지 확인하고, **빠진 단어는 wp 에 채워 넣는다**.
- "생성(translate) → 검증(verify wp coverage) → DONE" 의 2단계 흐름.

**구현 아이디어 (착수 시 확정)**:
- 파이프라인(`process_paper_prompt.md`)에 검증 단계를 명시: 각 `ss` 항목에 대해
  내용어 토큰을 추출 → `wp` 에 매칭되는지 확인 → 누락분 보강 → 재검증.
- `validate_paper.py` 확장: 문장별 **wp 커버리지 지표**(내용어 대비 wp 비율) 계산,
  임계치 미만이면 검증 실패로 떨어뜨려 워커가 재시도/보강하게.
  - 영어 토큰화(기능어 stopword 제외) → 각 토큰이 어떤 `wp[0]` 에 포함되는지 검사.
  - 한국어 뜻(`wp[1]`)이 실제 `ko` 문장에 등장하는지(substring)까지 확인하면 정확도↑.
- 워커 흐름: 검증 실패 시 claude 에게 "누락 단어 목록"을 주고 wp 보강 → 통과까지 루프(상한 N회).

**메모**: 이미 번역된 논문은 소급 적용 안 됨 → 재번역 시 검증 단계가 적용됨.
